## Что это за продукт

Локальный **control plane (панель управления) для параллельных кодинг-агентов**.
Инструмент для разработчика, который хочет запускать несколько AI-агентов
(сейчас — OpenAI Codex CLI) одновременно, каждого в своей полностью
изолированной среде, и управлять ими из одной веб-панели.

Каждый агент живёт в отдельной лёгкой виртуалке (OrbStack/Ubuntu VM), внутри
которой лежит копия рабочего репозитория и весь его dev-стек (Postgres, Redis,
Hasura, Temporal через docker compose). VM клонируются copy-on-write почти
мгновенно (~0.07–0.4s), поэтому "выдать нового агента" дёшево.

### Ключевые свойства / суть бренда

- **Параллелизм**: много агентов работают одновременно, каждый над своей
  задачей/веткой. Образ "рой / флот / армия агентов".
- **Изоляция**: каждый агент в своей VM, не топчет файлы хоста и других агентов.
- **Скорость**: мгновенное клонирование готовой базовой VM (CoW).
- **Оркестрация**: одна панель плодит, запускает, останавливает и сравнивает
  результаты агентов.
- **Для разработчиков**: это dev-инструмент/IDE-смежная утилита, не язык и не
  редактор сам по себе.

### Технический стек

- Frontend: React 19 + Vite, xterm.js (терминалы), кастомный diff-вьювер.
- Backend: Node + Effect-TS, HTTP + WebSocket, оборачивает `orbctl`.
- Среда агентов: OrbStack VM (Ubuntu), docker compose, OpenAI Codex CLI.

## Контекст бренда

- Компания-владелец: **KeenEthics**.
- Хочется, чтобы имя начиналось с **"Keen"** (наследование бренда компании).
- Рассматривается обыгрыш через домен/точку, напр. `keen.code`.
- **Компания часто использует тему космоса и астронавтов** — желательное
  направление метафор. Ложится естественно: продукт построен на OrbStack
  (рабочее имя `orb`), а архитектура напрашивается на космические образы:
  параллельные агенты = рой / флот / созвездие / экипаж; изолированные VM =
  капсулы / орбиты / спутники вокруг ядра-репозитория; сам control plane =
  **ЦУП / mission control / Houston**.

keenfleet

Флот кодинг-агентів, кожен у своїй ізольованій капсулі. Запускай, спостерігай і порівнюй десятки паралельних запусків з одного командного центру.

Як це працює Навіщо
навіщо

Один розробник — цілий екіпаж

Замість одного агента, який гальмує на кожній задачі, — флот. Кожен член екіпажу працює над своєю задачею чи гілкою, у повністю ізольованому середовищі, не заважаючи іншим.

🛰️

Паралелізм

Багато агентів працюють одночасно, кожен над своєю задачею. Не чекаєш — роздаєш.

🧪

Ізоляція

Кожен агент в окремій легкій VM з копією репо та повним dev-стеком. Не чіпають файли хоста й одне одного.

⚡

Швидкість

Базова VM клонується copy-on-write майже миттєво.

🎛️

Оркестрація з одного екрану

Список агентів, живі термінали, diff-в'ювер, кнопки start / stop / порівняти — усе в одній панелі.

архітектура

Капсули на орбіті навколо ядра

Веб-панель керує control-plane сервером, який оркеструє легкі віртуалки через OrbStack. Кожна VM — самодостатня капсула з агентом і його оточенням.

процес

Від кліку до результату

1

Тисни «+ new»

Панель клонує золоту базову VM — з усіма залежностями, образами та прогрітим кешем збірки.

2

Агент стартує в капсулі

Усередині VM піднімається dev-стек (docker compose up) і запускається Codex над копією репо.

3

Спостерігай із ЦУПу

Живий термінал, статус стеку, прогрес — усе видно в панелі. Відкрий VS Code Remote одним кліком.

4

Порівнюй і забирай

Дивись diff кожного агента, залишай найкращий результат, решту гаси.

економіка

Новий агент коштує майже нічого

Базова VM запікається один раз. Клони шарять диск через copy-on-write — тому масштабувати флот дешево.
